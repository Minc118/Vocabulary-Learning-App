# Project Log

这个目录专门用于保存项目过程记录，不再把这类内容持续堆叠到 `README.md` 里。

## 这个目录存什么

- 每次功能改动的修改日志
- 当次任务用到的知识点
- 关键技巧和踩坑记录
- 实现流程步骤
- 为什么这样做
- 当前方案的优缺点

## 记录规则

1. 每次较大的功能改动，新建一个按日期命名的 Markdown 文件。
2. 文件名建议格式：

```text
YYYY-MM-DD-主题.md
```

例如：

```text
2026-04-20-flask-rest-homework.md
```

3. `README.md` 以后尽量只保留：

- 项目简介
- 如何运行
- 技术栈
- 高层说明

4. 过程性学习记录、作业解释、实现细节、思路推演，统一写到这里。

## 推荐模板

每次记录建议包含这些部分：

- 背景 / 任务目标
- 本次修改了什么
- 涉及文件
- 用到的知识点
- 实现步骤
- 为什么这么做
- 优点
- 缺点 / 未完成项
- 下次可以怎么继续
