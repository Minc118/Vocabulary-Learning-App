# Skill: Pre-Commit Audit

## Purpose

Use this skill before committing changes generated by an AI agent.

The goal is to ensure the commit is clean, focused, safe, and does not include secrets, junk files, accidental deletions, or unrelated changes.

---

## Core Principle

**Do not commit until the diff is understood.**

The agent must inspect and report before creating a commit.

---

## Safety Rules

1. Do not run `git reset --hard`.
2. Do not run `git clean`.
3. Do not force push.
4. Do not stage all files blindly with `git add .` unless explicitly approved.
5. Do not commit `.env` files.
6. Do not commit secrets, tokens, passwords, database URLs, service keys, or personal credentials.
7. Do not commit unrelated generated files.
8. Do not commit tool scratchpad files unless intentionally part of the repository.
9. Do not push unless explicitly approved.

---

## Required Checks

Run:

```bash
git status
```

```bash
git diff --name-only
```

```bash
git diff --stat
```

If safe and useful, inspect the actual diff:

```bash
git diff
```

Check recent commits:

```bash
git log --oneline -5
```

---

## Secret Scan

Search for sensitive strings in changed files.

Examples:

```bash
grep -R "access_token\|refresh_token\|Authorization: Bearer\|service_role\|SUPABASE_SERVICE_ROLE\|GOOGLE_CLIENT_SECRET\|client_secret\|sk-\|E2E_TEST_PASSWORD=\|DATABASE_URL" .
```

The agent must not print actual secret values. If a match appears, report the file path and the type of risk only.

---

## Build / Test Checks

Depending on the project, run safe checks such as:

```bash
npm run build
```

```bash
python3 -m py_compile path/to/file.py
```

```bash
python3 -m unittest discover
```

```bash
curl http://127.0.0.1:<port>/api/health
```

If authenticated E2E testing exists, run it only with a dedicated test account and sanitized output.

---

## Staging Rules

Prefer explicit staging:

```bash
git add path/to/file1 path/to/file2
```

Avoid:

```bash
git add .
```

unless the user explicitly approves and all changed files are verified.

```text

Do not stage files yet.
Do not commit.
Do not push.
```
