> 本文档定义了页面结构、用户流程、导航结构和页面布局规范。

## 二、导航结构 (Navigationsstruktur)

### 左侧主导航（6 个主入口）

```
🏠 首页        Dashboard      /
📚 单词库      Vocabulary     /vocabulary
📥 导入        Import         /import
🔄 复习        Review         /review
📁 分类        Collections    /collections
📊 统计        Statistics     /statistics
```

底部可选：
```
⚙️ 设置        Settings       /settings
```

### 导航设计原则

| 原则 | 说明 |
|------|------|
| ❌ 不要把「添加单词」做一级导航 | 它是单词库内部动作 |
| ❌ 不要把「AI补全」做一级导航 | AI 只是辅助功能，不是用户目标 |
| ❌ 首页不要堆满所有信息 | 首页核心是看今日任务 + 快速进入关键动作 |

### 六个入口的职责

| 入口 | 回答的问题 |
|------|----------|
| 首页 | 今天先做什么？ |
| 单词库 | 我有哪些词？ |
| 导入 | 词从哪里来？ |
| 复习 | 怎么巩固？ |
| 分类 | 怎么整理？ |
| 统计 | 学得怎么样？ |

## 三、页面职责说明

### 1. Dashboard（首页）

**核心任务**：回答用户三个问题 — 今天要做什么？可以从哪里开始？最近学得怎么样？

**包含模块**：

| 区域 | 内容 |
|------|------|
| 顶部 | 欢迎语 / 今日日期 / 今日待复习数 |
| 中间主区 | 三个快速入口按钮：开始复习、添加单词、导入文章 |
| 下方 | 最近新增单词列表 + 学习统计摘要卡片 |

### 2. Vocabulary List（单词列表）

**核心任务**：管理所有词汇（找词 + 管词）

| 区域 | 内容 |
|------|------|
| 顶部 | 搜索框 + 新增按钮 |
| 筛选区 | 标签、分类、语言、熟练度、来源 |
| 主体 | 单词列表（卡片或表格视图） |

### 3. Word Detail（单词详情）

**包含**：单词本体、翻译、词性、解释、例句、搭配、标签、来源、复习状态与历史

**操作**：编辑按钮、删除按钮

### 4. Add / Edit Word（添加/编辑单词）

**布局**：

| 区域 | 内容 |
|------|------|
| 左侧 | 基础输入：单词、翻译、语言 |
| 右侧 | AI补全区：词性、解释、例句、搭配、近义词 |
| 底部 | 标签/分类选择 + 保存按钮 |

**逻辑**：基础信息在左，AI增强在右，最后统一保存

### 5. Import Step 1（粘贴文本）

| 区域 | 内容 |
|------|------|
| 主体 | 大文本框 + 语言选择 |
| 底部 | 「开始分析」按钮 |

### 6. Import Step 2（候选词确认）

| 区域 | 内容 |
|------|------|
| 左侧 | 文章上下文（分句高亮） |
| 右侧 | 候选词列表 + 勾选框 |
| 底部 | 「AI批量补全」按钮 |

### 7. Import Step 3（批量保存）

| 区域 | 内容 |
|------|------|
| 主体 | 批量补全结果预览（可展开查看每个词） |
| 侧栏 | 统一分类和标签选择 |
| 底部 | 「确认保存」按钮 |

### 8. Review Hub（复习中心）

**包含**：今日待复习数、预计时间、复习队列构成（新/学习中/复习）、最近表现

### 9. Review Session（复习会话）

**布局**：极简，专注复习

| 区域 | 内容 |
|------|------|
| 顶部 | 进度条 + 关闭按钮 |
| 中央 | 大卡片（问题面 → 答案面） |
| 底部 | 「显示答案」按钮 → 三档评分：不会 / 模糊 / 会 |

> 复习页的关键是专注 (Fokus)，不能信息过载。

### 10. Review Result（复习结果）

**包含**：本次完成数、会/不会比例、用时、明日待复习预估

### 11. Collections & Tags（分类与标签管理）

**包含**：分类列表、标签列表、新建按钮、查看某分类下全部词汇

### 12. Collection Detail（分类详情）

**包含**：分类名称、词汇数量、该分类下的词汇列表、可筛选/编辑/删除/开始复习

### 13. Statistics（统计）

**包含**：总词汇数、待复习数、最近7天新增趋势、最近7天复习趋势、分类分布

## 四、核心用户流程 (User Flows)

### 流程 A：手动添加单词

```
Dashboard
  → 点击「添加单词」
  → Add Word 页面
  → 输入单词 + 选择语言
  → （可选）输入翻译
  → 点击「AI补全」
  → 系统生成例句/词性/搭配等
  → 用户编辑确认
  → 添加标签/分类
  → 保存
  → 跳转到 Word Detail 或 Vocabulary List
```

**设计要点**：路径短、一次完成、AI是中间辅助不是强制步骤

### 流程 B：从文章导入单词

```
Dashboard
  → 点击「导入文章」
  → Import Step 1: 粘贴文本
  → 点击「开始分析」
  → Import Step 2: 候选词确认
  → 系统提取候选词
  → 用户勾选需要保存的词
  → 点击「AI批量补全」
  → Import Step 3: 批量保存
  → 用户统一加标签/分类
  → 保存
  → 跳转到 Vocabulary List
```

**设计要点**：先导入 → 再筛选 → 再补全 → 最后保存。不能一上来就自动全部保存，因为候选词里会有噪音。

### 流程 C：每日复习

```
Dashboard
  → 显示「今日待复习 N 个」
  → 点击「开始复习」
  → Review Session
  → 显示词卡问题面
  → 用户先回忆
  → 点击显示答案
  → 选择「不会 / 模糊 / 会」
  → 系统更新复习状态
  → 下一张
  → 完成后进入 Review Result
```

**设计要点**：单任务、节奏快、连续感强。复习最怕被打断，页面必须极简。

### 流程 D：按分类查找和整理词汇

```
Collections
  → 点击某个分类（如「德语B1面试」）
  → Collection Detail
  → 查看该分类下所有单词
  → 可筛选 / 编辑 / 删除 / 开始复习
```

### 流程 E：查看单词细节并再次编辑

```
Vocabulary List
  → 点击某个单词
  → Word Detail
  → 查看释义/例句/来源/复习记录
  → 点击编辑
  → Edit Word
  → 保存
```

**设计要点**：AI内容和用户理解会不断变化，词条必须可持续修正。

## 七、设计要点总结

### 核心设计原则

1. **首页只做任务分发，不做大杂烩** — 首页核心是入口，不是资料库
2. **单词库和复习必须分开** — 「管理词」和「学习词」是两种心智模式 (Mental Model)
3. **导入流程必须分步** — 天然是多阶段流程（三步向导 Wizard）
4. **AI不要单独做模块** — 用户目标不是「用AI」，而是「学词」
5. **每个词都要能回到详情页** — 词汇学习一定会发生反复修正
